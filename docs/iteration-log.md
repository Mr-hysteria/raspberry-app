# Iteration Log

> 用途：记录每次迭代做了什么、为什么做、影响了什么，以及下一位接手者需要注意什么。

## 使用规则

- 每次迭代结束后至少新增一条记录
- 记录重点放在：
  - 本次目标
  - 实际完成项
  - 影响范围
  - 遗留问题
  - 下一步建议
- 不要求写很长，但要让下一位 agent 能在 1 分钟内理解上下文

## 模板

复制下面结构新增下一条即可：

```md
## YYYY-MM-DD

### 目标

- 本次迭代目标

### 完成项

- 完成了什么

### 影响范围

- 影响到哪些模块、脚本、文档、运行方式

### 遗留项

- 暂未解决的问题
- 需要后续确认的风险

### 下一步建议

- 建议下一位 agent 或开发者继续做什么
```
