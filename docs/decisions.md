# Decisions Log

> 用途：记录关键技术决策，重点写清楚“背景、选择、原因、影响”。

## 记录规则

- 只有在出现重要取舍时才新增，不需要为所有小改动都写一条
- 一条记录应回答下面几个问题：
  - 当时遇到了什么问题
  - 备选方案有哪些
  - 最终选了什么
  - 为什么这样选
  - 带来了什么收益和代价

## DEC-002 每日一句使用后台请求和单文件缓存

### 状态

- 已采纳

### 日期

- 2026-06-24

### 背景

- 时钟需要展示金山词霸每日一句及其背景图
- 目标设备内存有限，且断网不能影响时钟主功能

### 最终决策

- 使用后台线程执行 HTTPS 请求
- 本地只保留一份 JSON 和一张 PNG
- 请求失败时继续使用上次成功缓存

### 原因

- 不阻塞 Slint 事件循环和每秒时钟刷新
- 避免引入数据库和复杂缓存管理
- 保证离线启动仍有内容可显示

### 代价与注意事项

- 首次从未联网时只能显示内置默认语录和纯色背景
- HTTPS 依赖会增加二进制体积和交叉编译时间

## DEC-003 使用 X11 DPMS 实现夜间息屏

### 状态

- 已采纳

### 日期

- 2026-06-24

### 最终决策

- 23:30–07:00 通过 `xset dpms force off` 息屏
- 触摸后通过 `xset dpms force on` 临时唤醒 60 秒

### 原因

- 复用当前 X11 运行环境，不引入硬件专属背光接口
- 与现有启动脚本中的 DPMS 配置保持一致

### 代价与注意事项

- 依赖 `DISPLAY`、`XAUTHORITY` 和 `xset`
- 不同触摸控制器在 DPMS 关闭后的事件传递行为需要真机验证

## DEC-001 从 `HTML + Chromium` 切换到 `Rust + Slint`

### 状态

- 已采纳

### 日期

- 2026-04-20

### 背景

- 目标设备是 `Raspberry Pi Zero 2 W`，只有 `512MB` 内存
- 原有 `HTML + Chromium` 方案实现快，但运行时成本较高，不利于长期稳定运行

### 备选方案

- 保持 `HTML + Chromium`
- 切换到其他原生 GUI 路线
- 切换到 `Rust + Slint`

### 最终决策

- 项目切换到 `Rust + Slint` 原生实现，并使用软件渲染方式运行

### 原因

- 降低内存占用
- 移除浏览器运行时依赖
- 减少首次运行弹窗和桌面浏览器副作用
- 更适合后续做开机直启和长期稳定运行

### 影响

- 代码结构从网页资源转为 Rust 源码 + Slint UI
- 运行方式从浏览器启动转为本地二进制启动
- 部署重心转移到 Rust 构建、交叉编译和树莓派运行脚本

### 代价与注意事项

- 团队需要维护 Rust / Slint 构建链路
- UI 调整需要重新编译，不像前端页面那样即改即看
- 后续每次 UI 或运行链路调整，都应同步维护文档资产
